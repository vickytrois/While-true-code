Trabalhando com links

Links externos

Um link externo aponta para um outro site. Como esse que leva para a meu perfil no GitHub.

Quando desenvolvemos um site, precisamos pensar como que a experiência de navegação vai contribuir para o comportamento dos usuários.

Por padrão, quando criamos um hyperlink, esse caminho é aberto na mesma janela que estava sendo navegada. Esse é um comportamento interessante para links internos, mas para links externos, pode gerar um resultado oposto ao que queremos (que os usuários permaneçam navegando no nosso site).

O atributo target

O atributo target nos ajuda a controlar como esses caminhos serão abertos e ele suporta os seguintes valores:

  • _self (abre o link na mesma janela ou frame e é a configuração padrão);
  • _blank (abre links em uma nova janela);

Existem os valores _top, _parent e o uso do <nome-do-frame>, mas são todos relacionados ao uso de frames e essa é uma técnica já à beira do desuso.

Links internos

Podemos usar links internos para ir de uma página para a outra em nossos sites. Como nesse exemplo que o link ir para página 02 nos manda para a página 02.

O atributo rel

Esse atributo nos ajuda a informar mecanismos de busca como a navegação no nosso site funciona.

Alguns valores aceitos pelo atributo rel são:

  • next (indica que a página referenciada é está à frente na ordem de navegação)
  • prev (indica que a página referenciada é antecessora na ordem de navegação)
  • author (indica que a página referenciada leva para o site do autor da página atual)
  • external (indica que o link referenciado não faz parte do seu site)
  • nofollow (indica que uma página não tem o seu aval para ser encontrada em mecanismos de busca, ou que ela está ali porque pagou para isso)
  • noopener e noreferrer (impedem que páginas mal intencionadas possam usar a página do nosso site como meio para phishing ou que a conexão com o servidor se torne muito pobre)

Links para páginas em subpastas de um projeto

Quando trabalhamos em projetos, é ideal que organizemos nossos arquivos em pastas, subpastas, seguindo uma lógica de organização que facilite a busca por recursos.

Para relacionar esses arquivos em páginas diferentes, precisamos digitar o caminho que deve ser percorrido até esses arquivos:

<a href="<nome da pasta>/<nome do arquivo>">texto visível na página</a>

Como aqui na nossa página de notícias.

Disponibilizando downloads

Para disponibilizar um arquivo para download, precisamos indicar o caminho para ele.

Crie uma pasta para o arquivo que você deseja disponibilizar, coloque seu arquivo dentro dela, se você desejar, você pode disponibilizar vários formatos, como um arquivo em extensão .pdf e outro em extensão .zip.

A marca <a> deve serguir esse modelo:

<a href="<nome da pasta>/<nome do arquivo>" download="<nome do arquivo>" type="<código do tipo do arquivo>">texto visível no site</a>

Para saber os códigos para cada tipo de arquivo acesse iana.org.
