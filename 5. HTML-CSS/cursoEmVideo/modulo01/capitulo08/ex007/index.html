Principais formatações

Forma semântica e não-semântica

De um tempo para cá, e com o avanço das CSSs e da preocupação de se desenvolver um código limpo, houve um grande foco na separação entre forma e significado.

HTML se detem ao significado e CSS à forma. No entanto, nem sempre foi assim. O HTML já fez uso de marcas que se atinham à forma e não ao conteúdo. Era o caso de <big>, <marquee> e <strong>, mas existem muitas outras.

O fato é que essas marcas foram descontinuadas pela 3WC e seu uso é desaconselhado.

Algumas marcas têm uma forma semântica e uma forma não semântica. Fique atento e dê preferência sempre para o uso da semântica, já que não há diferênça visível entre elas.

Negrito / Destaque

A forma não-semântica para formatarmos um termo em negrito é com a marca <b>, já a marca <strong> é sua forma semântica.

Itálico / Ênfase

No caso da forma itálica a marca <i> é a forma não-semântica e a semântica, a marca <em>.

Sublinhado / Inserido

E para sublinhados a marca <u> é do tipo não-semântico e <ins> do tipo não-semântico.

Texto marcado

A tag <mark> faz com que o fundo de um texto fique destacada como quando usamos uma marca-texto.

Seletores de estilo

Dentro da marca que se deseja formatar

Às vezes precisamos formatar uma marca poucas vezes, ou uma única vez. Nesses casos, podemos usar o seletor de estilo dentro da própria marca que desejamos formatar:

<mark>style="background-color: aqua;"</mark> , como feito na sessão acima.

No início do arquivo, abaixo da marca <title>

Porém, há estilos que se repetem demais ao longo de um texto ou página, mas não no projeto todo. Então, criar uma marca <style> no início do documento e dentro dela adicionamos seletores de formatação, como este:

h3{
 font-color:"<cor desejada>
}

Em folhas de estilo

Quando precisamos padronizar algumas formatações o indicado é que se faça uso de uma folha de formatação, recurso que vamos aprender mais adiante.
